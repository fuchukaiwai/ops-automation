# 府中界隈 (FuchuKaiwai)

府中を拠点に、インフラエンジニアがゆるく集まって「クラウドインフラ運用をもっと楽にする」ことを目指すコミュニティです。AWSを中心に、運用×自動化×AIをテーマに調査・検証・実装・共有を行います。

---

## 1. 名称
**府中界隈 (FuchuKaiwai)**  
府中発のインフラ運用コミュニティ。

---

## 2. 目的

- クラウドインフラ（主にAWS）の運用現場で発生している手作業・定型作業を自動化する
- AIを組み合わせて「人が毎回考えていること」を仕組みに落とす
- 得られた知見・コードをコミュニティの資産として残す
- 将来的には「AWS運用をAIで手助けする」形のアウトプット（本・公開リポジトリ）にする

---

## 3. 活動イメージ

1. **課題出し**：実運用で困っていること・手でやっていることを持ち寄る  
   - 例）ログ確認がつらい／バックアップ確認を人がしている／アラート一次対応が人依存 など
2. **調査・検証**：AWSサービス・OSS・AI（Bedrock, OpenAI など）で置き換えられるかを調べる
3. **実装**：スクリプト・Lambda・IaC・手順書ジェネレータなどで形にする
4. **共有・レビュー**：月内のどこかで実装内容を発表、コードはGitHubに蓄積
5. **再利用・公開の検討**：社外に出せる形に一般化するかを話す

---

## 4. 進め方

- **第2火曜：調査・実装の日**
  - 前回出たタスクを各自／ペアで進める
  - 技術的に詰まったところをその場で相談
- **第4火曜：発表・共有の日**
  - 作ったもの・試したことをLT形式で共有
  - 次回取り組むタスクを決める
- 初回は運営やルールのすり合わせを行う

※開催曜日を変更する場合はここを更新してください。

---

## 5. テーマ例

- AWS運用 × AI（ログ要約、メトリクス異常検知、アラート自動返信）
- バックアップ／リストアの自動チェック
- ウイルス対策・パッチ適用の自動レポート化
- リソース情報収集の自動化（タグ・構成差分）
- Slack / Discord への自動通知連携
- 手順書をコードにする（Runbook → Script）
- 初期構築時の運用用OSユーザー一括作成
- 初期構築時のディレクトリ作成および権限設定
- 初期構築時の運用スクリプト配布
- ドキュメントMCPサーバーの検証

## 9. ルールのたたき台

- 実運用の機密情報は持ち込まない（アカウントID・IP・顧客名など）
- 作ったものは原則コミュニティに共有する
- 発表は短く・定期的に
- 新しく来た人にもわかるように Discord のチャンネルを整理する
- 詳細なルールについては、"docs/"に記載
